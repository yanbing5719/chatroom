# 目前遇到的问题以及解决方法

## 2026.6.8
1.整体架构应该是服务器只负责设置好监听，连接客户端，不负责业务

2.变量应该初始化

3.每一个函数都应该检查他的返回值

4.epoll

```
epoll_wait()
↓
判断 fd
├─ listenfd → accept() 新客户端 → 注册到 epoll
└─ clientfd → 线程池处理消息 → Dispatcher → Manager → 发送/存储
```

5.join是强制等待线程结束

## 2026.6.9


1.声明类时应该先在.hpp文件里面写好框架，再到.cpp文件里面具体完善，头文件用到哪个写哪个不需要都写在.hpp或者.cpp文件里面

2.如果想要在另一个类里面用到其他类的私有变量，应该在那个类里面写一个变量值获取函数，然后再在要用这个变量的类里面直接写进参数里面，最后在调用时，将他的返回值，作为参数传进去

3.聊天的实现

**私聊**
```
+----------+          +----------+          +----------+
| Client A | -------> |  Server  | -------> | Client B |
+----------+          +----------+          +----------+
```

**群聊**
```
                          +----------+
                          | Client B |
                          +----------+
                                ^

                                |

+----------+          +----------+          +----------+
| Client A | -------> |  Server  | -------> | Client C |
+----------+          +----------+          +----------+
                                |
                                |
                                v

                          +----------+
                          | Client D |
                          +----------+
                        
```

4.epoll只能负责发现事件不进行任何处理,对于监听还是通信的文件描述符应该单独储存起来，最后再由其他函数统一处理

5.完成这个项目建议将高并发服务器整体封装起来，目前现在是一盘散沙
